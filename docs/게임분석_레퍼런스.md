## 1. Guild Master - Idle Dungeons 분석

### 1.1 게임 개요

| 항목 | 내용 |
|------|------|
| **개발사** | Paranoid Squirrels |
| **출시일** | 2024년 5월 22일 |
| **최종 업데이트** | 2025년 12월 1일 |
| **장르** | 방치형 던전 크롤러 RPG |
| **플랫폼** | Android, iOS |
| **평점** | 4.5/5 (Phone), 4.3/5 (Tablet) |

### 1.2 핵심 게임 시스템

#### 1.2.1 모험가 시스템

**기본 클래스 (4종)**
- **보병 (Infantry)**: 근접 전투 특화
- **견습생 (Apprentice)**: 마법 계열로 분화 (클레릭, 파이어 위자드, 리치 등)
- **궁수 (Archer)**: 원거리 물리 딜러
- **도적 (Thief)**: 민첩 기반 전투

**클래스 분화**
- 70개 이상의 세부 클래스 언락 가능
- 티어 성장 시스템: 최대 9단계까지 성장
- 레벨당 5레벨씩 상승

**모집 시스템**
- 선술집에서 무작위 모험가 출현
- 시간당 1명씩 추가 (선입선출 방식)
- 모집 비용 무료, 숙소 자리 필요
- 특성 시스템: 0~2개 특성 보유 (일부 과금 전용 특성 존재)

#### 1.2.2 던전 시스템

**지역 구성**
- 현재 11개 지역 (2025년 7월 기준)
- 지역당 최대 4명 모험가 + 1마리 펫 배치

**지역별 특징 예시**
| 지역 | 특징 | 공략 포인트 |
|------|------|-------------|
| 마법의 숲 | 황금 토끼(고회피), 숲의 정령(기절) | 민첩 43+, 공격력 40+ 필요 |
| 눈 와이번 던전 | 보스 근접공격 면역 | 궁수/법사 계열 강제 |
| 어둠 던전 | 어둠 80, 마방 100 골렘 | 성기사 강제, 물리 딜러만 |

**레이드 시스템**
- 일일 1회 도전 가능
- 추가 입장: 30보석 소모
- 에픽 레이드: 1회 클리어, 진행도 저장

#### 1.2.3 길드 관리

- 시설 건설 및 업그레이드
- 장비 관리 및 창고 시스템
- 다중 팀 편성 (역할별 최적화)

### 1.3 수익화 모델

#### 1.3.1 과금 요소

| 유형 | 내용 |
|------|------|
| **보석 (Gems)** | 프리미엄 재화 |
| **인벤토리 확장** | 필수적 과금 포인트 |
| **스타터팩** | 약 15,000원 (권장) |
| **추가 레이드 입장** | 30보석 |

#### 1.3.2 무과금 보상

- 주당 약 $3-4 상당의 보석 무료 획득 가능
- 광고 시청을 통한 보상

### 1.4 커뮤니티 반응

#### 1.4.1 긍정적 평가

```
✅ 관리형 방치 게임으로 적합
✅ 정기적인 업데이트
✅ 개발자의 적극적인 소통
✅ 전략적 깊이가 있는 팀 구성
✅ 광고 최소화
```

#### 1.4.2 부정적 평가

```
❌ 드롭률이 매우 낮음 (초반 아이템도 며칠 소요)
❌ 진행 속도가 극도로 느림
❌ 방치 게임인데 방치 패널티 존재 (경험치 손실)
❌ 인벤토리 확장 비용 급등
❌ 저장 공간 관리 스트레스
❌ 아이템 폐기 기능 부재
```

#### 1.4.3 대표 유저 의견

> "하루에 몇 번 체크하는 게임으로는 좋지만, 진행이 정말 느리다"

> "아이템 드롭률이 악몽이다. 첫 번째 지팡이(+4 지력)도 이틀이 걸렸다"

> "방치 게임이 방치를 벌하는 건 이상하다"

### 1.5 게임 디자인 인사이트

#### 장점 - 채택 고려
1. **클래스 분화 시스템**: 70+ 클래스로 다양성 제공
2. **팀 편성 전략**: 상황별 최적화된 팀 구성
3. **지역별 특화 전략**: 특정 직업/빌드 강제로 다양성 유도
4. **레이드 시스템**: 일일 콘텐츠로 복귀 유도

#### 단점 - 개선 필요
1. **극단적 드롭률**: 초반부터 답답함 유발 → 초반 진행감 개선 필요
2. **방치 패널티**: 장르 정체성 위배 → 보상만 있는 시스템 권장
3. **인벤토리 압박**: 과금 강요 느낌 → 기본 공간 충분히 제공

## 2. Habitica 분석

### 2.1 앱 개요

| 항목 | 내용 |
|------|------|
| **개발사** | HabitRPG, Inc. |
| **장르** | 게이미피케이션 습관/할일 관리 앱 |
| **플랫폼** | Android, iOS, Web |
| **사용자 수** | 110만+ |
| **가격** | 무료 (인앱 구매 선택) |

### 2.2 핵심 시스템

#### 2.2.1 과제 유형

| 유형 | 설명 | 보상/패널티 |
|------|------|-------------|
| **습관 (Habits)** | 반복 가능한 행동 (+/-) | 클릭마다 XP/골드 또는 HP 손실 |
| **일일 과제 (Dailies)** | 매일 해야 할 일 | 미완료 시 HP 손실 |
| **할 일 (To-Dos)** | 일회성 과제 | 오래될수록 보상 증가 |

#### 2.2.2 캐릭터 시스템

**레벨 시스템**
- 과제 완료 → 경험치(XP) 획득
- 미완료 → 체력(HP) 손실
- 레벨업 → 스탯 포인트 획득

**직업 시스템 (레벨 10 이상)**
| 직업 | 특징 |
|------|------|
| 전사 (Warrior) | 높은 HP, 파티 버프 |
| 힐러 (Healer) | 파티 회복 |
| 마법사 (Mage) | 경험치 획득 보너스 |
| 도적 (Rogue) | 골드 획득 보너스 |

**스탯 시스템**
- 힘 (STR): 데미지 증가
- 체력 (CON): HP 증가
- 통찰력 (PER): 아이템 드롭률
- 지력 (INT): 경험치 보너스

#### 2.2.3 소셜 시스템

**파티 (Party)**
- 친구와 함께 퀘스트 진행
- 보스전: 파티원의 과제 완료가 보스에게 데미지
- 파티원 미완료 시 전원 피해 (상호 책임감)
- 1인 파티도 가능

**길드 (Guild)**
- 관심사 기반 소셜 그룹
- 그룹 챌린지 참여
- 퀘스트는 불가 (파티만 가능)

**퀘스트 시스템**
- 퀘스트 스크롤로 시작
- 수집형 퀘스트 / 보스전
- 일일 과제 완료 = 퀘스트 진행

### 2.3 수익화 모델

#### 2.3.1 구독 옵션

| 기간 | 가격 | 할인율 |
|------|------|--------|
| 월간 | $4.99 | - |
| 3개월 | $14.99 | - |
| 6개월 | $29.99 | - |
| 연간 | $47.99 | 20% |

#### 2.3.2 보석 (Gems)

- 프리미엄 재화
- 펫, 장비, 커스터마이징 구매
- **비판점**: 골드→보석 전환 불가, 획득 매우 어려움

### 2.4 게이미피케이션 심리학

#### 2.4.1 작동 원리

```
과제 완료 → 즉각적 보상 (XP/골드/아이템)
    ↓
도파민 분비 → 성취감
    ↓
습관 형성 → 내재적 동기 강화
```

#### 2.4.2 효과 통계

- 기존 할일 앱 대비 **30% 높은 유지율**
- 게임 요소 도입 후 **37% 생산성 향상** (콜로라도 대학 연구)
- 직원 83%가 게이미피케이션 요소에 동기 부여됨 (TalentLMS)

#### 2.4.3 특히 효과적인 대상

- **ADHD 사용자**: 외부 보상 시스템이 과제 시작 장벽 극복에 도움
- **실행 기능 장애**: 게임화된 피드백이 즉각적 만족감 제공

### 2.5 커뮤니티 반응

#### 2.5.1 긍정적 평가

```
✅ 독창적인 RPG 스타일 습관 앱
✅ 높은 커스터마이징 자유도
✅ 무료로 충분히 사용 가능
✅ 크로스 플랫폼 지원 (웹/앱)
✅ 레트로 픽셀아트 디자인
✅ 파티 시스템의 상호 책임감
✅ ADHD/신경다양성 사용자에게 특히 효과적
```

#### 2.5.2 부정적 평가

```
❌ 한국어 미지원
❌ 동기화 문제 (웹-앱 간)
❌ UI가 복잡하고 배우기 어려움
❌ 보석 획득이 너무 어려움
❌ 2023년 커뮤니티 포럼 폐쇄
❌ 고객 지원 불만족
❌ 앱 버그/프리징 이슈
❌ 기업 환경에는 부적합
```

#### 2.5.3 역효과 연구 결과

**주의점**: 학술 연구에서 역효과도 발견됨

> "사용자의 49%만이 보상이 적절하다고 평가"

> "생산적인 시기에 오히려 페널티를 받는 경우 발생 (과제 체크 시간 부족)"

> "포인트에 집중하다 실제 습관 형성 소홀"

### 2.6 게임 디자인 인사이트

#### 장점 - 채택 고려
1. **즉각적 피드백**: 모든 행동에 시각적/수치적 보상
2. **사회적 책임감**: 파티 시스템으로 상호 동기 부여
3. **직업 시스템**: 플레이 스타일에 따른 보너스 차별화
4. **도전 과제/퀘스트**: 장기 목표 제공

#### 단점 - 개선 필요
1. **복잡한 온보딩**: 초보자 친화적 튜토리얼 필요
2. **역효과 가능성**: 패널티 시스템 완화 고려
3. **프리미엄 재화 접근성**: 무과금 사용자도 합리적 획득 경로 필요

## 3. 공통 인사이트 및 텅장히어로 적용 방안

### 3.1 방치형 게임 핵심 원칙

#### 3.1.1 리텐션 메커니즘

```
┌─────────────────────────────────────────────────┐
│              방치형 게임 리텐션 공식              │
├─────────────────────────────────────────────────┤
│  오프라인 수입    →   복귀 동기 부여              │
│  수입 상한선     →   정기적 접속 유도             │
│  빠른 초반 진행   →   초기 이탈 방지              │
│  장기 목표       →   지속적 플레이 동기           │
└─────────────────────────────────────────────────┘
```

#### 3.1.2 참여 시계 (Engagement Clocks)

| 주기 | 콘텐츠 예시 |
|------|-------------|
| 30분 | 자원 수집, 미니 보상 |
| 4시간 | 무료 뽑기, 일일 던전 |
| 24시간 | 일일 퀘스트, 출석 체크 |
| 주간 | 주간 레이드, 특별 보상 |

### 3.2 두 게임 비교 분석

| 요소 | Guild Master | Habitica | 텅장히어로 적용 |
|------|--------------|----------|-----------------|
| **핵심 루프** | 모험가 수집→던전 파밍 | 과제 완료→보상 획득 | 재무 습관→캐릭터 성장 |
| **진행 속도** | 매우 느림 | 적당함 | 초반 빠르게, 후반 느리게 |
| **소셜** | 없음 | 파티/길드 | 선택적 소셜 기능 |
| **패널티** | 경험치 손실 | HP 손실 | 최소화 권장 |
| **복잡도** | 높음 | 중상 | 점진적 언락 |

### 3.3 텅장히어로 적용 권장 사항

#### 3.3.1 Guild Master에서 채택

1. **클래스/직업 분화 시스템**
   - 재무 습관 유형별 캐릭터 분화
   - 예: 저축왕 → 투자가 / 절약러 / 자산가

2. **팀 편성 시스템**
   - 여러 저축 목표를 팀처럼 관리
   - 목표별 최적화된 전략 제안

3. **지역(던전) 시스템**
   - 재무 목표 = 던전
   - 달성 시 다음 목표(지역) 언락

4. **레이드 시스템**
   - 일일/주간 재무 챌린지
   - 큰 보상으로 복귀 유도

#### 3.3.2 Habitica에서 채택

1. **즉각적 피드백**
   - 저축 시 즉시 시각적 보상
   - 경험치 바, 레벨업 알림

2. **습관/일일/목표 구분**
   - 습관: 커피 안 사기, 대중교통 이용
   - 일일: 지출 기록
   - 목표: 비상금 100만원 모으기

3. **파티 시스템 (선택적)**
   - 가족/친구와 저축 챌린지
   - 상호 책임감으로 동기 부여

4. **직업 시스템**
   - 재무 스타일별 보너스
   - 예: 저축형/투자형/절약형

#### 3.3.3 피해야 할 요소

| 요소 | 이유 | 대안 |
|------|------|------|
| 느린 초반 진행 | 초기 이탈 | 튜토리얼 기간 가속 |
| 방치 패널티 | 스트레스 유발 | 보상만 있는 시스템 |
| 복잡한 온보딩 | 진입 장벽 | 점진적 기능 해금 |
| 과도한 과금 유도 | 이탈 + 악평 | 합리적 무과금 경험 |
| 인벤토리 압박 | 불편함 | 충분한 기본 공간 |

### 3.4 수익화 전략 제안

#### 3.4.1 권장 과금 모델

```
┌─ 기본 무료 (핵심 기능 모두 제공)
│
├─ 광고 시청 보상
│   └─ 자발적 시청 → 보너스 보상
│
├─ 선택적 프리미엄
│   ├─ 광고 제거
│   ├─ 추가 커스터마이징
│   └─ 프리미엄 테마/아바타
│
└─ 구독 (선택)
    └─ 추가 분석 기능, 클라우드 백업 등
```

#### 3.4.2 피해야 할 과금 요소
